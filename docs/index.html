01 — Features

Everything you need.
Nothing you don't.

1

Typo Catcher

Zero-latency after learning
Type gti stauts and dude suggests git status. After 3 corrections it stops asking the LLM entirely — instant recall from a local SQLite database.
2

Ask Anything

natural language
Just type what you want in plain English. “kill whatever is on port 3000” and dude gives you the command. No googling. Follow-ups work too — 15-minute session memory.
3

Pipe Anything

stdin analysis
Feed it logs, kubectl output, or error messages. cat crash.log | dude "wtf happened to gpu" and get an actual diagnosis, not a man page.
4

Learns Your Habits

Personalized suggestions
Analyzes your shell history to understand your tools, directories, and style. Suggestions are personalized, not generic Stack Overflow pastes.
5

Secret Filtering

Privacy by default
Environment variables containing KEY, TOKEN, SECRET, or PASSWORD are automatically redacted before anything is sent to the LLM.
6

Full Transparency

No hidden data
Run dude context "your query" to see exactly what gets sent to the LLM. System prompt, user context, history, session. Inspect everything.
02 — Safety

Before you hit Enter,
read what it says.

Dude suggests commands, not runs them behind your back. You always see what's about to execute. Three safety modes let you decide how much hand-holding you want — and even in “yolo” mode, the truly terrifying stuff is always blocked.

Always blocked — rm -rf /, dd if=/dev/zero, and friends. Every mode. No exceptions.
Confirm (default) — asks before every command. If you're paranoid, good. So are we.
Auto — safe commands run instantly (ls, git status). Everything else asks.
YOLO — never asks. But destructive commands are still blocked, because we care about you even if you don't.
safety demo
$ rm -rf /
dude: Nope. Blocked.
  This command would destroy your
  entire filesystem. Not even
  in yolo mode.

$ gti push --forc
